# Mission: Rhino Plugin Development

## Why

I want to design and build serious, production-quality plugins for Rhinoceros 3D. I need to understand custom UI creation, code architecture, and Rhino's data persistence model so I can ship plugins that feel professional and integrate deeply with Rhino's document model.

## Success looks like

- I can scaffold a new Rhino plugin project from scratch without a wizard
- I can build custom docking panels with rich WPF/Eto controls
- I can save and load plugin-specific data inside `.3dm` files using UserData
- I can structure a multi-command plugin with clean separation of concerns
- I can package and distribute a plugin via Yak

## Constraints

- Learning in evenings/weekends — lessons must be completable in under 30 minutes
- Windows + Visual Studio (assumed, but not yet confirmed)

## Out of scope

- Grasshopper component development (separate track)
- Rhino Compute or cloud services
- Rhino for Mac specific concerns
- C/C++ native plugin SDK (sticking to RhinoCommon / C#)
