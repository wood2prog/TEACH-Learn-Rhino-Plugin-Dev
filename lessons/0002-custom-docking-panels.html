Lesson 0002: Custom Docking Panels

Bring your plugin to life with a dockable UI

Tied to your mission: Custom user interfaces are one of your primary goals. This lesson teaches the foundation — a dockable, live panel that reacts to the Rhino document.

What you'll learn

  • The panel lifecycle: registration → creation → display → disposal
  • How to register a UserControl as a Rhino panel
  • How to add open/close commands for the panel
  • The EventWatcher pattern — keeping UI in sync with document changes
Prerequisite: You should have a working plugin project from Lesson 0001. If not, create one first.

The big picture

A Rhino panel is just a System.Windows.Forms.UserControl that Rhino wraps in a dockable container. The call chain:

YourPlugin.cs (OnLoad) │ └─> Panels.RegisterPanel(this, typeof(MyPanel), "Title", icon) │ └─> Rhino creates MyPanel instance when user opens the panel │ └─> MyPanel lives until Rhino closes (System panels) or document closes (PerDoc panels)

You provide the content. Rhino provides the docking frame, tabbing, and visibility toggling.

Step 1: Add a UserControl to your project

In your existing plugin project from Lesson 0001:

  1. Right-click the project → Add → Class
  2. Name it MyPanel.cs
  3. Replace the contents:
using System;
using System.Windows.Forms;
using Rhino;
using Rhino.DocObjects;

[System.Runtime.InteropServices.Guid("A1B2C3D4-E5F6-7890-ABCD-EF1234567890")]
public class MyPanel : UserControl
{
    private ListBox _objectList;
    private Label _countLabel;

    public MyPanel()
    {
        InitializeControls();
        SubscribeToDocumentEvents();
    }

    private void InitializeControls()
    {
        _countLabel = new Label
        {
            Text = "Objects: 0",
            Dock = DockStyle.Top,
            Height = 24,
            Padding = new Padding(4)
        };

        _objectList = new ListBox
        {
            Dock = DockStyle.Fill
        };

        Controls.Add(_objectList);
        Controls.Add(_countLabel);
    }

    private void SubscribeToDocumentEvents()
    {
        RhinoDoc.AddRhinoObject += OnObjectAdded;
        RhinoDoc.DeleteRhinoObject += OnObjectDeleted;
        RhinoDoc.ReplaceRhinoObject += OnObjectReplaced;
    }

    private void OnObjectAdded(object sender, RhinoObjectEventArgs e)
    {
        RefreshObjectList();
    }

    private void OnObjectDeleted(object sender, RhinoObjectEventArgs e)
    {
        RefreshObjectList();
    }

    private void OnObjectReplaced(object sender, RhinoReplaceObjectEventArgs e)
    {
        RefreshObjectList();
    }

    private void RefreshObjectList()
    {
        if (_objectList.InvokeRequired)
        {
            _objectList.Invoke(new Action(RefreshObjectList));
            return;
        }

        var doc = RhinoDoc.ActiveDoc;
        if (doc == null) return;

        _objectList.Items.Clear();
        foreach (var obj in doc.Objects)
        {
            _objectList.Items.Add(
                $"{obj.Id}: {obj.ObjectType}");
        }
        _countLabel.Text = $"Objects: {doc.Objects.Count}";
    }

    protected override void Dispose(bool disposing)
    {
        if (disposing)
        {
            RhinoDoc.AddRhinoObject -= OnObjectAdded;
            RhinoDoc.DeleteRhinoObject -= OnObjectDeleted;
            RhinoDoc.ReplaceRhinoObject -= OnObjectReplaced;
        }
        base.Dispose(disposing);
    }
}

Replace the [Guid] with your own — use Tools → Create GUID in Visual Studio.

Step 2: Register the panel in OnLoad

Open your MyFirstPluginPlugin.cs and add the registration call:

protected override LoadReturnCode OnLoad(ref string errorMessage)
{
    var panelType = typeof(MyPanel);
    Rhino.UI.Panels.RegisterPanel(
        this,                    // your PlugIn instance
        panelType,               // the UserControl type
        "My Panel",              // title shown in the tab
        null                     // icon (optional)
    );

    return LoadReturnCode.Success;
}
Note: The RegisterPanel overload shown here registers a per-document panel (the default). A new panel instance is created each time a document is opened. Use PanelType.System for a panel that survives across documents.

Step 3: Add open/close commands (optional)

Panels are accessible from Rhino's Panel menu, but it's often useful to have commands. Add a new command class:

using Rhino.UI;

[Guid("F1A2B3C4-D5E6-7890-ABCD-EF1234567890")]
public class ToggleMyPanelCommand : Command
{
    public override string EnglishName => "ToggleMyPanel";

    protected override Result RunCommand(RhinoDoc doc, RunMode mode)
    {
        var panelType = typeof(MyPanel);
        if (Panels.IsPanelVisible(panelType.GUID))
            Panels.ClosePanel(panelType.GUID);
        else
            Panels.OpenPanel(panelType.GUID);

        return Result.Success;
    }
}

Step 4: Run it

  1. Press F5 to launch Rhino
  2. Type ToggleMyPanel — your panel should appear
  3. Add a sphere, box, or any geometry — the panel list updates live
  4. Delete an object — the count decreases
Pro tip: You can also right-click any existing panel tab and check/uncheck "My Panel" in the context menu.

Understanding EventWatchers

The RhinoDoc class exposes static events for document changes. The three you need for a live panel:

Event Fires when
RhinoDoc.AddRhinoObject An object is added to the document
RhinoDoc.DeleteRhinoObject An object is deleted
RhinoDoc.ReplaceRhinoObject An object is modified (moved, deformed, etc.)
RhinoDoc.BeginOpenDocument A new or existing document is being opened
RhinoDoc.EndOpenDocument Document is fully loaded — refresh your panel
Threading trap: RhinoDoc events can fire from non-UI threads. Always check Control.InvokeRequired (WinForms) or Dispatcher.CheckAccess() (WPF) before touching UI controls in event handlers.

Exercise — make it interactive

Add selection tracking

  1. Subscribe to RhinoDoc.SelectObjects and RhinoDoc.DeselectObjects events
  2. When the selection changes, update a label in your panel with the count: "Selected: N"
  3. In the listbox, prefix selected objects with "* "

WPF and Eto alternatives

This lesson uses Windows Forms (simple, Windows-only). Alternatives available:

  • WPF — Same RegisterPanel call, but your control derives from System.Windows.Controls.UserControl. Richer styling, data binding. Windows-only.
  • Eto — Cross-platform (Windows + Mac). Derive from Eto.Forms.Panel. Use when targeting both platforms.
