Lesson 0001: Your First Rhino Plugin

Project anatomy and a running command

Tied to your mission: A serious plugin starts with a proper project. This lesson gets you from zero to a running command in Rhino.

"The only way to learn a new programming language is by writing programs in it." — Dennis Ritchie

What you'll learn

  • How to install the Rhino Visual Studio templates
  • The two essential classes every plugin needs: PlugIn and Command
  • How to run your command inside Rhino
  • How Rhino discovers and loads plugins

Before you start

You need Visual Studio 2022+ and Rhino 8 installed. Since you have both, we're good.

You'll also need the Rhino Visual Studio Templates extension. This is a VSIX that adds "RhinoCommon Plugin" as a project template.

The big picture

A Rhino plugin is a .NET class library (DLL) that Rhino loads at startup. Every plugin has exactly two required classes:

  1. PlugIn — the entry point. One per plugin. Manages lifecycle, document data, settings.
  2. Command — an invokable action. A plugin can have many commands.

That's it. All the complexity (UI panels, file IO, geometry) builds on top of this foundation.

Key insight: Commands are stateless. They receive a RhinoDoc, do work, and return a Result. State lives in the document or in your PlugIn class.

Step 1: Install the VS template

If you haven't already, install the Rhino Visual Studio Templates:

  1. In Visual Studio, go to Extensions > Manage Extensions
  2. Search for "Rhino"
  3. Install Rhino Visual Studio Templates by McNeel
  4. Restart Visual Studio

The template is available on the VS Marketplace. If you have difficulty with the in-IDE search, you can download it directly from the browser.

Step 2: Scaffold the project

  1. File > New > Project
  2. Search for "Rhino" — select RhinoCommon Plug-In
  3. Name it MyFirstPlugin and choose a location
  4. In the template dialog, accept defaults (General Utility, Rhino 8 target)
  5. Click Finish

The template generates two key files:

  • MyFirstPluginPlugin.cs — your PlugIn subclass
  • MyFirstPluginCommand.cs — your first Command subclass

Step 3: Understand the generated code

The PlugIn class

Open MyFirstPluginPlugin.cs. It will look like this:

// GUID and PlugInDescription attributes omitted for clarity
public class MyFirstPluginPlugin : PlugIn
{
    protected override LoadOnStartup GetLoadOnStartup()
        => LoadOnStartup.Whenever;

    protected override Result OnLoad(ref string errorMessage)
    {
        // Initialisation — runs when Rhino loads the plugin
        return Result.Success;
    }
}

Key points:

  • GetLoadOnStartup tells Rhino when to load the plugin. Whenever means "when a command is invoked."
  • OnLoad is your constructor-equivalent. Register panels, read settings, etc. here.
  • The [Guid] attribute on the class is how Rhino identifies your plugin across sessions and in .3dm files.

The Command class

Open MyFirstPluginCommand.cs. It will look like this:

public class MyFirstPluginCommand : Command
{
    public override string EnglishName => "MyFirstPlugin";

    protected override Result RunCommand(RhinoDoc doc, RunMode mode)
    {
        RhinoApp.WriteLine("Hello from MyFirstPlugin!");
        return Result.Success;
    }
}

Key points:

  • EnglishName becomes the Rhino command name. Users type MyFirstPlugin at the command line.
  • RunCommand receives the active RhinoDoc — never call RhinoDoc.ActiveDoc in a command.
  • Result tells Rhino whether to enable undo. Result.Success means "I did something undoable."
Common pitfall: If your command changes no document state, return Result.Success anyway. Returning Result.Nothing suppresses undo even if you did make changes.

Step 4: Run it

  1. Press F5 (Debug > Start Debugging)
  2. Visual Studio builds the plugin and launches Rhino
  3. In Rhino, type MyFirstPlugin at the command line and press Enter
  4. You should see: Hello from MyFirstPlugin! in the command history

That's it. You've built and run a Rhino plugin.

Step 5: Exercise — make it yours

Modify the command

  1. Change EnglishName to something short (e.g., "HelloRhino")
  2. Use RhinoApp.WriteLine to print the current document's name (doc.Name) and the number of objects in it (doc.Objects.Count)
  3. Use doc.Objects.AddSphere(...) to add a sphere at the origin with radius 5, then print its ID
  4. Build, run, type your new command name

Hint: Rhino.Geometry.Sphere takes a Point3d center and a double radius.

If you get stuck, the answer is a single Rhino command away — just ask.

How Rhino discovers plugins

Rhino scans several folders for .rhp files (renamed DLLs):

  • Your plugin's output directory (during development, via the VS debugger)
  • %AppData%\McNeel\Rhinoceros\8.0\Plug-ins\ (per-user)
  • C:\Program Files\Rhino 8\Plug-ins\ (system-wide)

When debugging with F5, Visual Studio copies the plugin to a special dev location and registers it with Rhino's plugin manager. No manual install needed during development.
